## ⭐ Proyecto Destacado — Reservi (Multi-Stack)

<div align="center">

### 🎯 [reservations-platform](https://github.com/DiegoMunoz212/reservations-platform)

**4 backends interoperables + 1 frontend, todo containerizado.**

[![Repo](https://img.shields.io/badge/Ver_en_GitHub-181717?style=for-the-badge&logo=github&logoColor=white)](https://github.com/DiegoMunoz212/reservations-platform)
![Stack](https://img.shields.io/badge/Multi--Stack-PHP_%7C_Go_%7C_Python_%7C_Java-A78BFA?style=for-the-badge)
![Languages](https://img.shields.io/badge/i18n-6_idiomas-F59E0B?style=for-the-badge)

</div>

Plataforma de reservas implementada en **4 lenguajes distintos** sirviendo
el **mismo contrato HTTP**, compartiendo **una sola Postgres** y **un JWT secret**.
Un token emitido por PHP es válido en Go, Python o Java — los clientes están
totalmente desacoplados de la implementación.

#### Qué demuestra el proyecto:

| 💡 | Lo aprendido / aplicado |
|---|---|
| **Multi-stack idiomático** | PHP/Laravel + Eloquent · Go/net/http + pgx · Python/FastAPI + asyncpg · Java/Spring MVC + JPA |
| **Concurrencia distribuida** | `pg_advisory_xact_lock(hashtext(professional_id))` dentro de transacciones — serializa solo escritores del mismo recurso, paralelismo total entre distintos |
| **JWT HS256 cross-backend** | Mismo secret entre los 4 stacks. Interop probado byte-por-byte con tests automáticos |
| **Frontend agnóstico** | Vue 3 + Vite + Tailwind con backend picker en vivo (healthchecks cada 10s) |
| **Internacionalización** | 6 idiomas: 🇪🇸 ES · 🇬🇧 EN · 🇧🇷 PT · 🇫🇷 FR · 🇰🇷 KO · 🇯🇵 JA |
| **Auth + roles** | JWT login/register + middleware admin para reportes |
| **DevOps** | Multi-stage Dockerfiles, docker-compose, Makefile, scripts bash de paridad |
| **Contract-first** | OpenAPI 3.0 como fuente de verdad, los 4 backends lo respetan |

#### Cómo levantarlo:

```bash
git clone https://github.com/DiegoMunoz212/reservations-platform.git
cd reservations-platform
make up        # postgres + 4 backends + frontend en contenedores
make test      # tests de paridad cross-backend
# → http://localhost:3000
```

> 🏗️ **~104 archivos · ~8.000 LoC** · construido en **~8 horas de pair-programming**
> con Claude (Anthropic) como par.

## 🧠 Conceptos que me apasionan

- **Contract-first design** — el contrato (OpenAPI/JWT) por encima de la implementación
- **Concurrencia honesta** — delegar la consistencia al motor de DB (transacciones + locks), no a código defensivo
- **Multi-stack interoperability** — un mismo dominio en varios lenguajes, drop-in replacements
- **i18n desde el día uno** — los textos viven en JSON, no hardcoded
- **DX > Magic** — herramientas claras (Makefile, scripts, healthchecks) sobre frameworks que esconden complejidad
